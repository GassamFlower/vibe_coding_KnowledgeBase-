# 门禁卡：业务门

> 包含阶段：业务模块开发（多轮）+ 安全审查  
> 过门禁标准：功能做完了，安全过了

---

## 过门禁标准（硬指标）

- [ ] 所有 P0 功能已开发完成
- [ ] 每轮开发都有验收证据
- [ ] 安全审查 5 道关卡全过
- [ ] 水平越权测试通过
- [ ] 无字符串拼接用户输入
- [ ] 安全审查报告已落盘
- [ ] 已 Git 提交

---

## 验收清单

### 业务模块开发（每轮）
- [ ] 本轮范围锁死（只做计划内的任务）
- [ ] 功能：核心路径走通（给执行命令和输出）
- [ ] 规则：无违反宪法 / 真源文档
- [ ] 安全：接口输入后端重新校验、权限校验到位、敏感数据哈希
- [ ] 数据：结构变更先改 schema
- [ ] 接口：返回示例覆盖成功 / 失败 / 边界
- [ ] 日志：关键操作有日志
- [ ] 回归：相邻功能未被破坏
- [ ] 真源文档已同步（如有规则变更）

### 安全审查
- [ ] 身份认证：哪些接口需要登录，在哪里校验
- [ ] 权限控制：哪些接口限定角色，无权限返回什么
- [ ] 输入校验：关键参数在哪里后端重新校验
- [ ] 数据归属：水平越权测试通过（用户 A 不能访问用户 B 的资源）
- [ ] 注入防护：无字符串拼接用户输入
- [ ] 密码规则齐全（长度 + 字符组合 + 弱密码拦截 + 哈希 + 失败限制）
- [ ] 无过度防御 / 死代码
- [ ] 安全审查报告已落盘
- [ ] 整改清单全部处理

## 常用 Prompt（3 个）

### 1. 模块开发计划（挖掘型）
```
当前阶段：业务模块开发 - [模块名]

本模块要做的接口 / 页面：[列]
依赖的已完成模块：[列]

请先输出本模块实施计划：
1. 接口清单（路径 / 方法 / 入参 / 出参 / 权限 / P0-P2）
2. 页面清单（路由 / 组件 / 状态 / P0-P2）
3. 涉及的业务对象 / 表变更
4. 涉及的规则变更
5. 本轮拆分（一个模块拆 N 轮）
6. 每轮的验收标准

我确认计划后再开始写。
```

### 2. 完成前验收（验收型）
```
在说"已完成"之前，请按以下清单自检并交证据：
- 功能：核心路径走通了吗？给执行命令和输出
- 规则：有没有违反宪法 / 真源文档？
- 安全：接口输入后端重新校验了吗？权限校验在哪里？
- 数据：数据库变更先改 schema 了吗？
- 接口：返回示例覆盖成功 / 失败 / 边界了吗？
- 日志：关键操作有日志吗？
- 回归：相邻功能没被破坏吧？给证据

每项给文件 / 命令 / 输出 / 日志作为证据。
未验证项标记"未验证"。不允许写"基本完成"。
```

### 3. 安全审查（验收型）
```
当前阶段：后端安全审查。不要只说"已做安全处理"。
你的角色：安全审查员。

请按 5 道关卡分别说明：
1. 身份认证
2. 权限控制
3. 输入校验
4. 数据归属（重点查水平越权）
5. 注入防护

每道关卡给：风险 / 处理位置（文件 + 行号）/ 处理规则 / 如何验证 / 是否通过。
最后输出整改清单，按严重 / 风险 / 建议分级。
```

## 常见遗漏项

- ⚠️ 一口气写完所有业务（必须拆轮）
- ⚠️ 跳过每轮验收
- ⚠️ 改了规则不改真源文档
- ⚠️ 一个模块未验收就开下一个
- ⚠️ 安全只做"要不要登录"忽略水平越权
- ⚠️ 让 AI 自己写过滤规则代替参数化查询
- ⚠️ 整改时顺手改其他代码
- ⚠️ 安全审查不给证据
