# 门禁卡：上线门

> 包含阶段：上线前总验收 + 上线 + Skill 反哺  
> 过门禁标准：验收过了，Skill 反哺了

---

## 过门禁标准（硬指标）

- [ ] 5 维度审查完成，无 🔴 严重项
- [ ] 🟡 风险项已登记后期清单
- [ ] 上线前验收报告已落盘
- [ ] 项目已上线
- [ ] 复盘报告已落盘
- [ ] 至少反哺 1 条新经验到 Skill 库
- [ ] 已 Git 提交

---

## 验收清单

### 上线前总验收
- [ ] 结构：目录放置 / 分层 / 重复实现
- [ ] 规则：宪法 / 真源文档 / 框架规范是否被违反
- [ ] 安全：输入校验 / 越权 / 密码 / 注入 / 过度防御
- [ ] 数据：schema / 范式 / 敏感数据 / 软删除
- [ ] 可维护性：命名 / 硬编码 / 提前封装 / 日志 / 错误 / 测试
- [ ] 无硬编码（配置 / URL / 密钥 / 错误信息）
- [ ] 无 TODO / FIXME 未处理
- [ ] 无死代码
- [ ] 环境变量都从配置读

### 上线
- [ ] 环境变量已配置
- [ ] 数据库迁移已执行
- [ ] 域名 / SSL 已配置（如需）
- [ ] 上线后观察清单已列（关键指标 / 告警阈值 / 回滚条件）

### Skill 反哺
- [ ] 踩坑清单已整理
- [ ] 至少 1 条新经验反哺到 Skill 库
- [ ] 复盘报告已落盘
- [ ] 下次项目提醒已记录

---

## 常用 Prompt（3 个）

### 1. 上线前总验收（验收型）
```
当前阶段：上线前总验收。本轮不改代码，只审查。
你的角色：代码审查员 + 上线把关人。

请对整个项目做 5 维度审查：
1. 结构：目录放置、分层、重复实现
2. 规则：宪法、真源文档、框架规范
3. 安全：输入校验、越权、密码、注入、过度防御
4. 数据：schema、范式、敏感数据、软删除
5. 可维护性：命名、硬编码、提前封装、日志/错误/测试

每项给"通过 / 风险 / 严重"判断 + 证据（文件 + 行号）。

风险分级：
🔴 严重：阻断上线
🟡 风险：建议修，可上线后修
🟢 建议：可选优化

输出整改清单。
```

### 2. 上线决策（验收型）
```
基于审查结果，请给出上线决策：
1. 总体评估：go / no-go / conditional go
2. 🔴 严重项清单（必须修完才能上线）
3. 🟡 风险项清单（可上线后修）
4. 🟢 建议项清单（可选优化）
5. 上线前必做动作
6. 上线后观察清单

给明确判断，不要"建议视情况而定"。
```

### 3. Skill 反哺（挖掘型）
```
当前阶段：项目上线 + Skill 反哺。
你的角色：项目复盘员。

请回顾本次项目：
1. 踩了哪些坑？（列具体场景 + 根因）
2. 哪些坑是现有 Skill / Prompt 没覆盖的？
3. 哪些 Skill 的规则需要强化或补充？
4. 下个项目开始前，应该更新哪些文档？

给具体建议：改哪个 Skill 文件、加哪条规则。不要泛泛而谈。
```

---

## 常见遗漏项

- ⚠️ 跳过总验收直接上线
- ⚠️ 🔴 严重项没修完就上线
- ⚠️ 上线完就结束，Skill 没迭代
- ⚠️ 反哺只写"以后注意"（要写具体规则）
- ⚠️ 不记录下次项目提醒
- ⚠️ 审查不给证据
- ⚠️ 顺手重构审查范围外代码
