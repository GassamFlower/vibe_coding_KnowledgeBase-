# 门禁卡：架构门

> 包含阶段：前端骨架 + 数据库设计 + 后端骨架 + 后端验收  
> 过门禁标准：四条线跑通了，真源文档有了

---

## 过门禁标准（硬指标）

- [ ] 前端能启动、首页能打开
- [ ] 数据库能连接、能建表、能插入查询
- [ ] 后端 4 条线跑通（启动线 / 接口线 / 业务线 / 运维线）
- [ ] 后端架构实施真源文档已落盘
- [ ] 已 Git 提交（至少 4 次：前端骨架 / 数据库 / 后端骨架 / 后端验收）

---

## 验收清单

### 前端骨架
- [ ] 设计风格已明确
- [ ] 设计 token 已定义（颜色 / 字号 / 间距 / 圆角 / 阴影）
- [ ] UI 组件库已接入
- [ ] 目录结构符合规范
- [ ] 路由结构搭好
- [ ] 接口层封装完成
- [ ] 项目能启动、首页能打开

### 数据库设计
- [ ] 业务对象清单已整理
- [ ] 对象关系已确认（ER 图）
- [ ] 表结构符合三大范式（或反范式有说明）
- [ ] 金额用 decimal、密码哈希存储、软删除字段一致
- [ ] schema.sql 已生成且可独立执行
- [ ] 数据库能连接、能建表、能插入查询测试数据

### 后端骨架
- [ ] 项目架构设计文档已落盘
- [ ] 目录责任表清晰
- [ ] 接口响应规则（11 种场景）
- [ ] 错误处理 / 日志 / 数据库连接规范
- [ ] 4 条线跑通（启动 / 接口 / 业务 / 运维）
- [ ] 配置 / 环境变量规范处理

### 后端验收
- [ ] 9 项验收每项有证据（文件 / 命令 / 输出 / 日志）
- [ ] 无"基本完成"等模糊词
- [ ] 《后端架构实施真源文档》已落盘
- [ ] Agent 宪法已加引用条款

## 常用 Prompt（3 个）

### 1. 前端骨架搭建（执行型）
```
现在按文档搭骨架。本轮范围锁死：
- 项目能启动
- 首页能打开
- 设计 token 已定义并生效
- UI 组件库已接入
- 目录结构按文档创建
- 路由结构搭好
- 接口层封装好

不要写业务页面。不要一次性做完所有功能。
完成后按验收清单自检，每项给证据。
```

### 2. 数据库设计（挖掘型）
```
当前阶段：数据库设计。先出设计文件，确认后再建表。
你的角色：数据库架构师。

请按以下顺序推进，每步等我确认再进下一步：
第 1 步：业务对象清单 + 关系图（Mermaid ER 图）
第 2 步：核心表设计（字段 / 类型 / 约束 / 索引）
第 3 步：三大范式校验
第 4 步：生成 schema.sql

每步给完让我 review，确认后再进下一步。
```

### 3. 后端验收（验收型）
```
当前阶段：后端架构验收。本轮不改代码，只验收。
你的角色：后端架构审查员。

请按 9 项验收清单逐项检查，每项必须给证据（文件 / 命令 / 输出 / 日志）。
未验证项标记"未验证"。禁止使用"基本完成""应该没问题"。

每项给：规则来源 + 实际证据 + 是否通过。
```

## 常见遗漏项

- ⚠️ 设计 token 没定就开始写页面
- ⚠️ 直接连数据库建表（必须先出 schema.sql）
- ⚠️ 金额用 float（必须用 decimal）
- ⚠️ 密码明文存储
- ⚠️ 后端骨架跳过架构文档直接写代码
- ⚠️ 4 条线没跑通就宣布完成
- ⚠️ 验收不给证据，只说"已完成"
- ⚠️ 真源文档写设计意图不写实际实现
- ⚠️ 改了规则没同步真源文档
