# Skill: browser-verification（浏览器验证）

## 何时使用

- 前端页面 / 交互 / 路由改动后；
- 用户说"帮我看看页面""打开看看效果""验证一下前端"；
- 涉及响应式、跨端单位（如小程序 rpx）问题时。

## 解决什么问题

避免 AI 只看代码就说"前端做好了"。让前端验收落到看得见的实际渲染、交互、状态反馈上。

## 执行流程

1. **启动本地服务**：确认本地服务能起来，记录端口、启动命令、健康检查结果。
2. **打开页面**：用浏览器打开目标页面，记录 URL、加载状态、控制台报错。
3. **检查视觉**：
   - 整体设计风格是否符合实施计划？
   - 设计 token（颜色、字号、间距、圆角、阴影）是否从统一变量取？
   - 主题规则是否生效？
   - 有没有硬编码样式导致的视觉不一致？
4. **检查响应式**：
   - 桌面尺寸正常？
   - 平板尺寸不乱？
   - 手机尺寸不挤成一团？
5. **检查交互**：
   - 按钮点击有反馈？
   - 表单提交失败有提示？
   - 加载中有状态？
   - 删除前有确认？
   - 路由跳转正常？
6. **检查跨端单位**（涉及小程序时关键）：
   - HTML 画布的 px 和小程序 `app.wxss` 的 px 不等价；
   - 48px 在手机上约等于 96rpx；
   - 必要时把 px 改成 rpx 对齐设计稿；
   - 圆角、按钮高度、卡片内边距等是重灾区。
7. **检查组件复用**：同类 UI 出现超过 2 次是否封装成组件？是否还在到处手写？
8. **截图留证**：关键页面截图，作为验收证据。
9. **输出验收报告**：覆盖以上各项，给"通过 / 待修"判断 + 截图位置。

## 输出

- 前端验收报告：URL / 视觉 / 响应式 / 交互 / 跨端单位 / 组件复用 / 截图位置。

## 约束

- ❌ 不在没启动服务的情况下说"前端做好了"；
- ❌ 不只看代码不看实际渲染；
- ❌ 不忽略控制台报错；
- ❌ 跨端项目不直接照搬 Web 单位（px 不等于小程序 px，更不等于 rpx）；
- ❌ 不忽略"相似 UI 出现 2 次以上未封装"的问题。
