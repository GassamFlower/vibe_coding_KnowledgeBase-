# Skill: backend-security-review（后端安全审查）

## 何时使用

- 后端骨架验收通过、开始写业务接口；
- 涉及登录、注册、修改密码、支付、订单、删除、权限相关功能；
- 上线前最后一次安全检查；
- 用户说"帮我做安全审查""看看有没有漏洞""这接口安全吗"。

## 解决什么问题

防止 AI 用"已经做了安全处理"敷衍。把风险、规则、处理位置、验证证据一项项交出来。看不懂的安全说明本身就不合格。

## 执行步骤

1. **拆 5 道安全边界**：

   | 关卡 | 含义 |
   |---|---|
   | 身份认证 | 这个请求是谁发的、有没有登录 |
   | 权限控制 | 这个人能不能做这件事 |
   | 输入校验 | 前端传进来的数据可不可信 |
   | 数据归属 | 这条数据是不是属于他自己 |
   | 注入防护 | 用户填的内容会不会被后端当命令执行 |

   让 AI 按接口分别说明：风险、处理位置、处理规则、如何验证、哪些还没验证。

2. **审查接口输入（默认不可信）**：列出关键输入（价格、数量、角色、用户 ID、订单归属、数据状态）和普通输入，前端校验只防体验，真正影响业务结果的必须后端重新判断。
3. **审查账号和密码规则**：注册 / 登录 / 修改密码 / 重置密码 / 创建管理员账号分别在哪里校验；密码长度 + 字符组合 + 弱密码拦截 + 失败次数限制；密码是否哈希存储；管理员账号是否有更严格要求。
4. **输出系统权限设计表**：每个接口说明是否需要登录、允许哪些角色、是否只能操作自己的数据、管理员是否可访问、权限判断在哪个文件/中间件、无权限返回什么、是否有测试证据。重点防两种越权：
   - 垂直越权（普通用户调管理员接口）
   - 水平越权（合法用户动了不属于自己的数据）
5. **检查注入风险**：SQL 注入、数据库查询注入、命令注入、HTML/模板注入。重点确认：用的是框架/数据库自带成熟方案，还是 AI 临时写的过滤代码。
6. **审查过度防御（死代码）**：必须严校验的业务规则（是否登录 / 是否有权访问 / 是否能修改 / 金额库存业务规则）必须严；输入框提示、格式提醒、前端展示优化不该复杂化。任何新增安全封装必须说明解决什么真实风险。
7. **审查配置文件安全**：
   - 密钥隔离：真正的密钥（数据库地址、数据库密码、登录令牌密钥、第三方接口密钥、管理员初始密码、支付或通知相关密钥）绝不能出现在代码里
   - 示例占位：示例配置（.env.example）只告诉别人"这里该填什么"，不暴露真实密钥
   - 环境注入：代码读配置时，应该从环境变量或框架推荐的方式里取，而不是直接写在代码里
   - 密钥重置：如果写入真实密钥到代码，必须换掉
8. **审查日志安全**：
   - 准许记录范围：哪些能记（请求日志、错误日志、数据库日志）
   - 禁止审计明细：哪些不能记（密码、token、密钥、手机号、身份证号、支付信息等敏感数据）
   - 敏感项脱敏处理：敏感内容有没有脱敏
   - 给出一条正常请求日志和一条错误日志示例
9. **审查依赖安全**：
   - 列出框架、库及对应版本
   - 核查已知安全漏洞和公告
   - 评估升级建议与功能影响
   - 锁定核心网络/加密/数据库依赖
10. **审查数据库安全**：
    - 数据资产分类：明确读/写/删接口清单（哪些是重要数据、哪些接口会读取/修改/删除它）
    - 四重安全验证：每个操作是否经过身份校验、权限校验、数据归属校验和业务规则校验
    - 参数传参防御：是否存在直接根据前端传入ID修改或删除数据的情况（严禁）
    - 机制合规要求：是否有软删除、事务或审计日志要求
    - 多张表一起变动时，必须用事务
11. **测试验证**：
    - 接口自动化测试：正常请求示例、参数错误示例、未登录示例、无权限示例、访问他人数据示例
    - 数据库一致性校验：数据库写入或修改前后的证据
    - 日志记录证据：日志记录证据
    - 多维度角色验证：普通用户、管理员、系统管理员、异常请求的人
    - 测试代码验收标准：功能可用、错误拦截、权限管控、边界校验、数据一致、日志追溯
    - 没有实际验证的项目标记为"未验证"
12. **整改 + 回归**：把"未验证"和"风险"项整理成清单，逐项修复后重新跑步骤 1-11。
13. **触发 Git 提交**：通过后提交一次。

## 输出

- 后端安全边界表；
- 系统权限设计表；
- 注入风险审查报告；
- 安全整改清单。

## 约束

- ❌ 不接受"已经做了安全处理"作为结论；
- ❌ 不只做"要不要登录"忽略水平越权；
- ❌ 不让 AI 自己写过滤规则代替参数化查询 / ORM；
- ❌ 不为了安全感堆一堆永远走不到的判断分支；
- ❌ 不把"理论上安全"作为通过证据；
- ❌ 不忽略密码明文存储、浮点金额、未哈希密码等问题。
