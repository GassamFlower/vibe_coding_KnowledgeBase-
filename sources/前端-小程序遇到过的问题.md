# 前端-小程序遇到过的问题

## 根本原因：HTML 画布用 px（CSS像素），小程序 app.wxss 也用 px（逻辑像素），但两者 1:1 不等价

属性 HTML 画布 小程序 app.wxss 实际渲染差异 按钮高度 48px 8px + 14px字体 + 8px ≈ 34px 小程序按钮矮 30% 按钮圆角 9999px （胶囊型） 6px （几乎直角） 胶囊变方块 按钮内边距 0 32px 8px 16px 小程序按钮窄 卡片圆角 24px （radius-xl） 10px （radius-md） 小程序卡片方 卡片内边距 32px 14px 小程序卡片挤 输入框高度 48px 11px + 14px + 11px ≈ 36px 小程序输入框矮 阴影 0 4px 8px 0 2px 12px 差异不大

核心结论 ：HTML 画布中 48px 在手机上约等于 96rpx ，而小程序里按钮实际只有 34px （约 68rpx ）。整个小程序的组件尺寸都偏小，圆角体系从"胶囊型"变成了"近似直角"。

## 修复方案

需要把 app.wxss 中所有组件的 px 值改为 rpx ，对齐 HTML 画布。要我直接动手改吗？改动集中在 app.wxss 的按钮、卡片、输入框、标签等组件样式，大约 20 处数值调整。
