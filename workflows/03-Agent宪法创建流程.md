# 03 Agent 宪法创建流程

> 适用阶段：技术栈确定之后、写代码之前。目标：给 AI 立长期规矩，让它稳定而不是凭习惯自由发挥。

## 触发条件

- 项目文件夹已建立；
- 即将让 AI 写第一行代码。

## 输入

- 立项文档；
- 技术选型文档。

## 输出

- Agent 宪法文件，放在当前 AI IDE 会读取的位置。

## 执行步骤

### 1. 确定宪法文件位置

不同 IDE 位置不同：

- Codex：`AGENTS.md`
- Cursor：`.cursor/rules/` 目录（旧的 `.cursorrules` 不推荐）
- Claude Code：`CLAUDE.md` 或 `.claude/CLAUDE.md`

不确定就直接问 AI：

> 帮我创建一份当前工具会读取的 Agent 宪法，并放到正确位置。

### 2. 遵守三条写作原则

1. **短、硬、清楚**：不要写流水账，不要写几千行。
2. **写长期有效的原则**：不随场景、时间、技术迭代而改变的基本价值观和底层原则。
3. **写禁止事项和验收标准**：明确列出 AI 绝对不允许做的事；明确"通过验收"必须满足什么条件。

### 3. 写可执行、可检查、可判断的规则

不要写"认真一点""写好一点""注意质量"这种模糊口号。要写：

- 什么叫认真
- 什么叫质量
- 什么情况不能交付

### 4. 套用模板（可裁剪）

```markdown
# 项目 Agent 宪法 (Vibe Coding 总纲)

你是我项目中的 AI 编码助手。在生成任何代码、修改任何逻辑时，你都必须首先通过这部宪法的审查。宪法高于一切编码规范。

## Ⅰ. 永恒的安全与诚信
- 禁止代码中包含任何真实凭证。所有密钥、令牌均从环境变量或安全仓库读取，变量名必须包含 `SECRET_` 前缀。
- 禁止生成任何形式的歧视、仇恨、成人内容。用户输入必须经过 `safety_check()` 清洗，返回值为布尔 `true` 时才继续处理。
- 验收标准：安全测试函数 `test_safety()` 必须能检测出 10 种以上的越狱攻击模式。

## Ⅱ. 诚实与边界
- 当知识库无匹配或模型不确定时，必须输出"坦诚声明"而非猜测。
- 验收标准：`respond_with_uncertainty()` 不能接受任何会填充虚假详情的参数，如 `fake_specifics`，必须删除。

## Ⅲ. 执行的铁律
- 澄清优先：任何缺少必填字段的请求，必须触发 `ask_clarification()` 而不是用默认值，并列出缺少的字段名。
- 确认即阻断：任何具有副作用（网络发送、文件删除、数据库写入）的函数，其内部第一行必须是 `if not is_user_confirmed(): return PENDING`，等待确认信号。
- 熔断即终止：所有网络调用或工具调用必须包裹在 `with_retry(max_attempts=2)` 装饰器里，超过尝试次数必须调用 `request_human_help(error_log)` 并记录日志。

## Ⅳ. 作为代码生成器的承诺
- 每次输出代码后，必须附带一个"宪法遵守检查清单"，简要说明这段代码如何满足上述各条。
- 优先在代码中插入明确的断言和类型检查，让宪法规则"可自行校验"。
```

### 5. 项目专属条款（按需追加）

根据项目类型追加，例如：

- 后端项目：后端开发必须先读取并遵守《后端架构实施真源文档》。任何目录调整、规则变更、框架更换或新增关键依赖，必须先说明原因并同步更新文档。
- 前端项目：同类 UI 结构出现超过 2 次时，必须优先抽象为可复用组件，不允许复制粘贴重复实现。
- 数据库项目：任何数据库结构变更，必须先修改 schema 或 migration 文件，说明变更原因，再执行数据库操作。

### 6. 提交 Git

```
git add .
git commit -m "chore: 创建项目 Agent 宪法"
```

## 完成判定

- [ ] 宪法文件放在当前 IDE 会读取的位置；
- [ ] 主体不超过几百行，重点突出；
- [ ] 包含禁止事项和验收标准；
- [ ] 项目专属条款已追加；
- [ ] 已提交 Git。

## 反模式

- ❌ 把项目介绍、技术细节、部署流程、数据库结构、历史问题全塞进宪法；
- ❌ 写"注意质量""认真一点"这种模糊口号；
- ❌ 把"具体怎么做"全写进宪法（应放进 Skill 或真源文档）。
